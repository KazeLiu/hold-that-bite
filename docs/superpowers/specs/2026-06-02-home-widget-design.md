# 桌面小组件第一版设计

## 目标

新增一个 `4x2` 桌面小组件，让用户不打开 App 也能看到今天的吃饭时间、禁食时间，并快速完成 `拒绝零食 +1`、`守住了`、`记录体重` 这些高频动作。

第一版只做 `4x2` 尺寸。后续再扩展 `2x2`、`4x1` 等多尺寸适配。

## 展示

- 小组件标题显示 `守住这口`。
- 主信息显示今天的吃饭时间，例如 `吃饭 09:00 - 17:00`。
- 辅助信息显示禁食开始提示，例如 `17:00 后禁食`；如果最后一口时间跨天，使用现有 `次日 HH:mm` 文案。
- 底部显示操作按钮：
  - `零食 +1`
  - `守住了`
  - `体重`
- 当设置里的 `体重趋势` 未开启时，隐藏 `体重` 按钮。

## 行为

- 点击 `零食 +1`：直接给今天增加一次拒绝零食记录，并刷新小组件。
- 点击 `守住了`：
  - 如果当前仍处于进食窗口内，打开 App，并显示鼓励弹窗：`我知道你快成功了，再坚持坚持`。弹窗可自动消失，也保留确定按钮。
  - 如果当前已进入禁食窗口，直接记录今天 `守住了`，然后刷新小组件。
- 点击 `体重`：仅在 `体重趋势` 开启时显示。点击后打开 App 的体重记录入口。
- 点击小组件非按钮区域：打开 App 主页。

## 架构

使用原生 Android App Widget：

- 新增 `AppWidgetProvider` 负责接收小组件生命周期和按钮点击事件。
- 新增 widget metadata XML，声明 `4x2` 的默认尺寸与可调整范围。
- 新增 RemoteViews 布局 XML，使用项目现有品牌色和简洁按钮样式。
- 新增轻量状态构建逻辑，从 `BiteStore` 读取设置，并根据 `FastingPlan` 与第一餐时间生成展示文案。
- 按钮行为优先复用现有快捷入口的 action 语义，避免同一业务分散成两套规则。

## 刷新

- 小组件添加、系统触发更新、按钮点击后刷新。
- App 内修改 `减肥安排`、`第一餐时间`、`体重趋势` 后刷新小组件。
- `拒绝零食 +1` 或 `守住了` 成功写入后刷新小组件。

## 错误处理

- 如果读取设置失败，使用 `AppSettings` 默认值展示 `16+8` 与 `09:00` 第一餐。
- 如果用户关闭了体重趋势，widget 立即隐藏 `体重` 按钮。
- `守住了` 的时间判断沿用现有 `FastingPlan.isEatingWindow`，避免 widget 与 App 主流程出现不同规则。

## 验证

- 单元测试覆盖 widget 展示模型：
  - 默认 `16+8` 文案。
  - 体重趋势关闭时隐藏体重按钮。
  - 体重趋势开启时显示体重按钮。
  - 跨天最后一口时间保留 `次日 HH:mm`。
- 编译 Android APK。
- 如果 MuMu 模拟器端口已开启，编译后安装到 MuMu 模拟器。
