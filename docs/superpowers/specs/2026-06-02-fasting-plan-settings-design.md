# 减肥安排设置设计

## 目标

在设置页允许用户选择限时进食安排，并设置第一餐时间。应用根据安排自动计算最后一口时间，帮助用户把“守住这口”的打卡标准从固定 16+8 扩展为可配置计划。

## 功能范围

- 支持 `16+8`、`14+10`、`12+12`、`18+6`、`20+4` 五种安排。
- 每个安排展示解释文案，例如 `8个小时吃东西，16个小时不吃东西`。
- 第一餐时间默认 `09:00`，用户可通过时间选择器修改。
- 最后一口时间由第一餐时间加上进食窗口自动计算；跨日时显示 `次日 HH:mm`。
- 用户选择和时间写入本地设置，重启后保留。

## 数据与交互

新增 domain 模型负责计划枚举和时间计算，避免把计算散落在 Compose UI 中。`AppSettings` 保存当前计划和第一餐时间；`BiteStore` 继续使用 `SharedPreferences` 持久化设置。设置页新增一个卡片，包含下拉计划选择、第一餐时间按钮和只读结果。

## 验证

新增单元测试覆盖默认安排、文案、普通时间计算和跨日时间计算。实现后运行相关单元测试与 Android 编译。
