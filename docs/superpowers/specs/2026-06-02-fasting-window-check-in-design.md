# 禁食窗口打卡展示设计

## 目标

让首页操作更贴合限时进食逻辑：进食窗口内不急着做最终打卡，禁食窗口内再判断今天是否守住；忘记打卡的历史日期默认按守住展示，但不自动写入记录。

## 规则

- 今天处于进食窗口内：只展示 `拒绝零食 +1`，隐藏 `守住了 / 没守住`，提示最后一口时间后回来记录。
- 今天处于禁食窗口内：展示 `拒绝零食 +1`、`没守住`、`守住了`。
- 过去日期没有真实打卡记录：日历和摘要按 `默认守住` 展示，不写入 `BiteStore`。
- 选择过去日期时：展示 `没守住 / 守住了`，用户可以回头覆盖默认状态。
- 未来日期：不展示最终打卡按钮。

## 实现

新增轻量 domain 逻辑判断当前是否处于进食窗口，并判断某天是否应按默认守住展示。首页继续复用现有打卡按钮组件，但由新的展示规则决定是否渲染；记录函数从“只能记录今天”扩展为“记录指定日期”。
