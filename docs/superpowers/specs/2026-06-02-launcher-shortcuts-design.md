# 桌面快捷入口设计

## 目标

让用户长按桌面 App 图标时，可以快速进入最常用的本地记录动作：拒绝零食、守住了打卡、记录体重。

## 入口

- `拒绝零食 +1`：静态 shortcut，始终显示。进入 App 后给今天拒绝零食次数加一，并回到首页选中今天。
- `守住了`：静态 shortcut，始终显示。进入 App 后如果今天已经到了可展示“守住了”的时间，就按今天守住了打卡；如果仍在进食窗口内，只弹出“我知道你快成功了，再坚持坚持”的鼓励窗，鼓励窗可点确定，也会自动消失。
- `记录体重`：动态 shortcut。只有体重趋势设置开启后才发布；关闭体重趋势后不出现在桌面长按快捷入口里。触发后进入趋势页并打开记录体重表单。

## 实现约束

- Android shortcut 的 icon 字段必须使用 drawable 资源，不能直接使用 emoji 文本；因此使用彩色 drawable 图标承载入口语义，shortcut 文案不再额外添加 emoji。
- `拒绝零食` 与 `守住了` 使用静态 XML shortcut，保证 App 启动前也能显示核心入口。
- `记录体重` 依赖用户设置，使用 `ShortcutManager.setDynamicShortcuts` 按当前设置发布或清除，并使用最高 rank，避免部分桌面只展示前几项时被挤掉。
- 快捷入口 intent 只消费一次，避免 Compose 重组重复执行记录动作。

## 验证

- 单元测试覆盖 action 解析、未知 action 忽略、体重入口动态发布规则。
- 编译 debug APK，安装到 MuMu 设备后通过 `dumpsys shortcut` 检查静态入口注册状态。
