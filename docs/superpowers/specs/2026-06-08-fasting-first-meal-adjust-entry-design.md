# 禁食期第一餐临时调整入口设计

## 目标

用户在默认第一餐时间前打开首页时，仍能临时调整今天的第一餐时间，而不必等到进食窗口面板出现。首页按自然日拆成第一餐前、进食窗口内、最后一口后三段，不把早晨第一餐前误当成最终打卡时间。

## 范围

- 首页今天处于第一餐前时，展示 `还没开始今天第一餐` 面板，不展示 `守住了 / 没守住`。
- 第一餐前面板提供 `现在开始` 和 `调整`。`现在开始` 会把今天临时第一餐设置为当前时间；`调整` 复用现有 `TodayFirstMealTimeDialog`。
- 如果用户把第一餐改到当前时间或更早，首页立即切换为现有进食窗口提示面板。
- 今天最后一口之后才展示 `守住了 / 没守住`。
- 选择过去日期时仍展示最终打卡按钮，方便补记；未来日期不展示最终打卡按钮。
- 设置页和今日临时调整都不接受会把最后一口时间推到次日的第一餐时间；切换计划导致越界时，把第一餐调整到该计划允许的最晚同日时间。
- 无效时间用不可自动消失的确认弹窗提示，标题为 `进食时间段不建议设置在午夜`，正文说明 AHA 科学声明对晚间/睡前进食的健康风险提醒，并给出当前计划允许的最晚第一餐时间。

## 验证

- 运行相关 domain 单元测试，确认时间窗口逻辑未被破坏。
- 编译 debug APK，确认 Compose 代码可编译。
- 编译成功后检查 MuMu 模拟器端口，若已开启则安装 debug APK。
